03 — Components

Six Core Components
Explained

COMPONENT 01
Layers

A neural network is organized into sequential layers — groups of neurons that process information at the same depth. The input layer receives raw data, hidden layers extract increasingly abstract features, and the output layer produces the final prediction.

Adding more layers increases a network's ability to learn complex patterns — this is the foundation of "deep" learning.

Input Layer → [Hidden Layers] → Output Layer
STRUCTURE
COMPONENT 02
Neurons

A neuron (or node) is the fundamental unit of computation. Each neuron receives inputs, applies weights and a bias, sums the result, then passes it through an activation function. It mimics how biological neurons fire when a signal crosses a threshold.

More neurons per layer = more capacity to represent complex patterns, but also more risk of overfitting.

output = activation(Σ(wᵢ · xᵢ) + b)
COMPUTATION
COMPONENT 03
Weights

Weights are the learnable parameters of a neural network — numerical values that determine how strongly one neuron's output influences the next. Initially set randomly, they are continuously updated during training.

A high positive weight amplifies a signal; a negative weight inhibits it. Training is essentially the process of finding the best possible weights.

w ← w − α · ∂L/∂w (gradient update)
LEARNING
COMPONENT 04
Activation Functions

Activation functions introduce non-linearity, enabling the network to learn complex, non-linear relationships. Without them, a deep network would behave like a single linear function, no matter how many layers it has.

Different activations suit different tasks: ReLU is fast and common for hidden layers; Sigmoid/Softmax are used in output layers for classification.

ReLU Sigmoid Tanh Softmax
NON-LINEARITY
COMPONENT 05
Loss Functions

The loss function (or cost function) measures how wrong the model's predictions are compared to the true labels. It returns a single number: a lower loss means better performance.

Common choices: Mean Squared Error (MSE) for regression tasks; Cross-Entropy Loss for classification. The loss is the signal that drives all learning.

MSE = (1/n) · Σ(yᵢ − ŷᵢ)²
ERROR SIGNAL
COMPONENT 06
Optimization Algorithms

Optimizers use the loss signal to update weights via backpropagation and gradient descent. The learning rate controls how large each update step is — too large and training is unstable; too small and it converges slowly.

Modern optimizers like Adam adapt the learning rate per parameter, combining the benefits of momentum and RMSProp for faster, more stable training.

04 — Summary

Why Visualizing
Neural Networks Matters

Neural networks are often treated as opaque "black boxes." Visualizing their architecture — layers, weights, activations, and learning curves — transforms them from mysterious functions into understandable systems. This transparency is essential for debugging, improving, and trusting AI models.

Hands-on experimentation with the Neural Network Playground reinforced how each component interacts: more hidden layers capture richer patterns, while noisy data demands regularization, and the choice of activation function can make or break convergence.

🧠

Depth = Capacity

More layers allow the network to learn hierarchical, abstract representations of data.

⚖️

Weights Encode Knowledge

All that a trained network "knows" lives in its weight values — learned through gradient descent.

📉

Loss Guides Learning

Without a clear error signal, a network cannot improve. Loss functions are the compass of training.

🔄

Iteration Is Everything

Training is a repeated cycle of predict → measure → adjust. Hundreds of thousands of times.

⚡

Activation Unlocks Complexity

Without non-linear activations, even a 100-layer network collapses to a single linear function.

🎯

Architecture Choices Matter

Selecting the right number of layers, neurons, and activation functions significantly impacts performance.
